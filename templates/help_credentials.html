Créer le fichier client_secret.json

Ce fichier est l'identifiant OAuth de l'application. Il permet à yt-poster de demander l'autorisation de poster sur tes chaînes YouTube. À créer une seule fois dans Google Cloud, puis à déposer ci-dessous (il est partagé par toutes les chaînes).

{% if has_secrets %} ✓ client_secret.json déjà présent sur le serveur {% else %} ✗ client_secret.json absent — suis les étapes ci-dessous {% endif %}
1

Crée un projet Google Cloud

Va sur console.cloud.google.com, connecte-toi avec le compte Google qui gère tes chaînes, puis crée un projet (menu en haut → « Nouveau projet »). Donne-lui un nom, ex. yt-poster.

2

Active les API nécessaires

Dans API et services → Bibliothèque, recherche et clique « Activer » sur :

  • YouTube Data API v3 — pour poster les vidéos (obligatoire)
  • YouTube Analytics API — pour les stats par jour et l'analyse des heures
3

Configure l'écran de consentement OAuth

Dans API et services → Écran de consentement OAuth : type « Externe », renseigne un nom d'appli et ton e-mail.

⚠️ Passe l'écran en « Production » (et non « Test »). En mode Test, l'autorisation des chaînes expire au bout de 7 jours et le planificateur s'arrête de poster. Si l'app reste en Test, ajoute au minimum ton compte dans « Utilisateurs tests ».

4

Crée l'identifiant « ID client OAuth »

Dans API et services → Identifiants → Créer des identifiants → ID client OAuth. Type d'application : « Application Web ».

Il faut bien « Application Web », pas « Application de bureau ». Le bouton « Connecter » de cette interface ne fonctionne qu'avec un client Web.
5

Ajoute l'URI de redirection autorisée

Toujours dans l'écran de création, section « URI de redirection autorisés », colle exactement cette adresse (déduite de ta configuration base_url) :

{{ redirect }}

Base actuelle : {{ base_url }}

Si tu changes de domaine plus tard (ex. passage en HTTPS / prod), reviens ajouter la nouvelle URI ici, sinon la connexion échouera.
6

Télécharge le JSON

Clique « Créer », puis sur l'identifiant fraîchement créé, bouton « Télécharger le JSON » (icône ⭳). Tu obtiens un fichier du type client_secret_XXXX.json.

7

Dépose-le ici

Importe le fichier ci-dessous. Il sera enregistré sous le nom client_secret.json et utilisé par toutes les chaînes. Ensuite, va sur chaque chaîne et clique « Connecter ».

Importer le client_secret.json

Le fichier doit contenir une clé web (client « Application Web »).
