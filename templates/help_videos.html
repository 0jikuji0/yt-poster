Ajouter des vidéos à une chaîne

Une fois une chaîne connectée (voir la doc client_secret.json), il suffit de lui donner des vidéos : le planificateur les publiera tout seul, une par une, aux heures configurées. Chaque vidéo peut avoir son propre titre / description / tags.

🎬Méthode simple : le dépôt web

C'est la méthode recommandée pour quelques vidéos à la fois.

1

Ouvre la chaîne, section « Déposer des vidéos »

Clique sur une chaîne dans la barre latérale. Sur sa page, la carte « Déposer des vidéos » contient une zone ＋ Fichiers vidéo + .json : clique dessus pour choisir un ou plusieurs fichiers (sélection multiple possible).

Formats acceptés : .mp4 · .mov · .avi · .mkv · .webm · .flv · .m4v

2

Renseigne les métadonnées communes

Sous la zone de dépôt, les champs s'appliquent à toutes les vidéos déposées qui n'ont pas leur propre fichier JSON :

TitreVide = le nom du fichier est utilisé comme titre.
DescriptionTexte libre. Laissé vide (et sans tags), la chaîne pioche dans son réservoir de hashtags configuré.
TagsSéparés par des virgules, ex. clip, best-of, 2026.
Confidentialitépublic, unlisted (non répertorié) ou private.
Contenu pour enfants« Made for kids » YouTube. Laisse sur Non sauf obligation légale.
3

Clique « Ajouter à la file »

Les vidéos rejoignent la file d'attente de la chaîne. Le planificateur les postera automatiquement dans la fenêtre horaire réglée, ou tu peux forcer avec « Poster maintenant ». Une vidéo déjà postée n'est jamais republiée.

Astuce : tu peux aussi déposer des vidéos depuis la page Bibliothèque, qui regroupe la file de toutes les chaînes.

🏷️Un titre différent par vidéo : les paires .json

Pour donner à chaque vidéo son propre titre/description/tags (au lieu des champs communs), dépose à côté de la vidéo un fichier JSON du même nom : clip01.mp4 + clip01.json. clipstudio les associe par leur nom de base.

Contenu type d'un fichier .json :

{
  "title": "Mon meilleur clip 😂",
  "description": "Abonne-toi pour plus !\n#shorts #gaming",
  "tags": ["shorts", "gaming", "best-of"],
  "privacy": "public",
  "made_for_kids": false
}
Seul title est vraiment indispensable. privacy et made_for_kids sont optionnels (valeurs par défaut de la chaîne sinon). Un JSON invalide est ignoré et les champs communs du formulaire prennent le relais.
Tu peux déposer en une fois un lot de paires (clip01.mp4, clip01.json, clip02.mp4, clip02.json…) : chaque vidéo récupère son propre JSON. Un .json déposé seul met à jour les métadonnées d'une vidéo déjà en file portant le même nom.

📦En masse : beaucoup de vidéos d'un coup

Pour des dizaines de vidéos, le dépôt navigateur reste possible (sélection multiple), mais devient lourd. Sur une installation serveur, on peut aussi copier les fichiers directement dans le dossier de la chaîne, sans passer par l'interface.

1

Structure d'un dossier de chaîne

Chaque chaîne a son dossier channels/<nom>/. On y dépose les vidéos (et éventuellement leurs JSON du même nom) :

channels/ma_chaine/
├── clip01.mp4
├── clip01.json   (optionnel — sinon titre = nom du fichier)
├── clip02.mp4
└── ...

Les vidéos sans JSON suffisent : clipstudio crée le sidecar automatiquement (titre = nom du fichier).

2

Copie les fichiers

Sur le serveur, copie ton dossier de vidéos dans channels/<nom>/. En déploiement Docker, l'équivalent est un docker cp vers le volume de données du conteneur, puis le planificateur prend le relais.

Après la copie, les vidéos apparaissent dans la file de la chaîne au prochain rafraîchissement de la page. Aucune action supplémentaire n'est requise.
Rappel quotas : ~100 vidéos/jour côté quota API, mais une chaîne récente ou non vérifiée est bridée par YouTube (~15–20/jour). Le planificateur étale les posts pour rester sous ces limites — inutile de tout vouloir poster le même jour.
